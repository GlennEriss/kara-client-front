## Notifications – Générer identifiant (V2)

### Optionnel

- **Notification au membre** : après réinitialisation du mot de passe, on peut envisager d’envoyer un email (ou une notification in-app) au membre pour l’informer que son mot de passe a été réinitialisé. Le PDF contenant les identifiants est en principe remis par un autre canal (remise en main propre, envoi sécurisé).
- **Notification admin** : non requis pour cette fonctionnalité (l’admin est l’acteur qui déclenche l’action).
- Si un tel flux est implémenté, documenter ici le canal (email, template, déclencheur) et le lien avec le module notifications du projet.
